# CorpRide — Telegram-бот заказа корпоративного водителя

Внутренний сервис для компании: сотрудники бронируют поездки с корпоративным
водителем через Telegram Mini App, водители видят расписание и заказы,
администраторы верифицируют доступ и управляют системой. Полное ТЗ —
[docs/tz.md](docs/tz.md).

> Репозиторий: https://github.com/MrDea1h/telegramm-orders-taxi
> Хостинг под прод пока не выделен — до тех пор весь код разрабатывается и
> проверяется локально.

## Статус проекта

Сейчас в репозитории — **демо-прототип дизайна и интерфейсов** ([webapp/](webapp/)),
согласованный перед началом реализации бэкенда/бота. Прототип:

- собран на реальном стеке финального Mini App (React 18 + TypeScript + Vite +
  Tailwind CSS + Framer Motion + Zustand), поэтому не выбрасывается — экраны
  дорабатываются на месте, когда подключается настоящий API;
- работает полностью на мок-данных, без бэкенда;
- переключает роль (Сотрудник / Водитель / Админ) и тему (светлая/тёмная,
  имитация Telegram `themeParams`) через демо-панель сверху — в проде роль и
  тема будут приходить от Telegram и бэкенда, а не выбираться вручную;
- карта — стилизованная SVG-заглушка (без API-ключей и внешних тайлов);
  реальный рендер карты (Yandex Maps JS API / MapLibre GL) подключается на
  этапе интеграции с гео-провайдером;
- Lottie-иллюстрации заменены на лёгкие анимированные SVG (Framer Motion) —
  финальные ассеты подключаются отдельно.

### Согласованные на старте решения (см. раздел 9 ТЗ)

| Вопрос | Решение |
|---|---|
| Оплата поездок | Полностью бесплатно для сотрудников, стоимость не учитывается |
| Карпулинг | Не нужен: 1 заказ = 1 водитель эксклюзивно на весь слот |
| Live-трекинг геопозиции водителя | Не нужен в MVP: только текстовые статусы (выехал / на месте / …) |

Гео-провайдер (Yandex Maps API / 2GIS / self-hosted OSRM) и политика на случай
отсутствия единственного водителя — открытые вопросы, не блокируют прототип
дизайна, будут закрыты перед стартом реализации бэкенда.

## Структура репозитория

```
docs/           долгоживущая документация (ТЗ и решения по ходу проекта)
webapp/         Mini App: сейчас — демо-прототип дизайна на мок-данных
```

Каталоги под бэкенд (`bot/`, `api/`, `worker/` и т.д.) появятся на этапе MVP
(Этап 1 по ТЗ), после согласования прототипа.

## Запуск прототипа

```bash
cd webapp
npm install
npm run dev
```

Откроется на `http://localhost:5173` (или другом свободном порту). Приложение
рендерится в рамке телефона на десктопе и на весь экран на мобильных.
